# POS - Structured Perceptron 词性标注系统

这是一个基于**结构化感知机（Structured Perceptron）**的中英文词性标注演示项目，提供 Web 交互页面。  
系统同时实现了一个 **HMM 基线模型**，用于和结构化感知机进行准确率对比。

## 功能

- 支持中文、英文词性标注。
- 核心模型：结构化感知机（序列标注，Viterbi 解码）。
- 对比模型：监督训练 HMM。
- Web 页面交互输入输出：输入文本、选择语言、选择模型、可视化显示 token/tag。
- 页面展示中英文测试集上的模型准确率与优缺点说明。

## 项目结构

- `app.py`：基于 Python 标准库 `http.server` 的服务入口，处理前端交互与模型调用。
- `pos_models.py`：结构化感知机、HMM、语料与评估逻辑。
- `app.py` 内置了 Web 页面模板与交互逻辑。

## 运行方式

```bash
python -m venv .venv
source .venv/bin/activate
python app.py
```

浏览器打开：`http://127.0.0.1:8000`

## 使用说明

1. 在文本框输入句子：
   - 中文建议输入分词结果（例如：`我 爱 自然 语言 处理`）；
   - 若中文不分词，系统会按字切分；
   - 英文可直接输入句子。
2. 选择语言（中文 / English）。
3. 选择方法（结构化感知机 / HMM 基线）。
4. 点击“开始标注”查看结果。

## 方法比较说明

- **结构化感知机**
  - 优点：判别式训练、可加入丰富特征、序列级解码。
  - 缺点：依赖特征工程，低资源情况下对未登录词泛化有限。
- **HMM 基线**
  - 优点：实现简单、参数可解释。
  - 缺点：独立性假设强，通常精度低于结构化判别模型。

> 备注：本仓库内置了小规模教学语料用于演示流程，适合课程作业展示与方法对比。
